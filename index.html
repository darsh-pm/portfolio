Projects 02

Task Board

Kanban · Productivity

Live Demo GitHub
Problem Teams lose alignment when task priority, status, and context live in different places. This board puts everything in one view — priority, due dates, subtasks, and AI recommendations.
Built Full-stack Kanban with drag-and-drop columns, per-card detail modals (title, description, priority, due date, subtasks with progress tracking), overdue filtering, and AI-powered task prioritization via Groq that reads open tasks and recommends what to focus on first.
Improvements Started with a simple title + due date card. Mid-build, realized that wasn't enough to be genuinely useful. Added a full detail modal — editable fields, description, subtasks with progress tracking. Took longer. Worth it.
Tradeoff Planned an activity log per card, deprioritized it. Adding it meant another database table and storage records — useful polish, not a new skill. Documented it as a planned feature and moved on.
Flow
React UI→ Supabase Auth + RLS→ PostgreSQL→ Groq API
Frontend Auth + DB AI Layer Tailwind

Pulse

PM Tool · Feedback Analysis

Live Demo GitHub
Problem Feedback from user interviews, support tickets, app reviews, and surveys lives in different places. PMs waste time aggregating instead of analyzing. Pulse centralizes it with structured categorization so patterns surface faster.
Built Feedback collection app with source/category/sentiment tagging, status tracking (Open → In Progress → Addressed), category breakdown with visual bars, multi-filter system, and AI analysis via Groq that identifies top pain points, what users love, and what to prioritize fixing first.
Complexity The multi-filter system with simultaneous criteria (sentiment + status + source + category) and the category breakdown component. Getting AND logic right across four independent filters took multiple iterations.
Roadmap Connect it to the Task Board so addressing a feedback item creates a linked task. Add a web survey form so feedback flows in directly rather than being entered manually.

A debugging story 03

One night both apps were loading extremely slowly or not at all. I opened the browser console expecting a code error.

What I found instead: Supabase auth token refresh requests were timing out. The code was fine. The database was fine. After isolating variables — switching from Wi-Fi to mobile data — everything loaded instantly.

The issue was network routing. My Wi-Fi couldn't reliably reach Supabase's auth endpoints. Mobile data could.

The fix wasn't in the code. It was understanding that the stack includes infrastructure I don't control, and knowing how to isolate which layer is failing.
